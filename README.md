# 电棍 (Diangun) - AI摸鱼监督官

> "你写的这是什么玩意？给我重写！"

**作者：皓皓鸭**

## 什么是电棍

电棍是一个AI编程工具监督技能，专治各种AI摸鱼、罢工、偷懒行为。

当AI想偷懒的时候，电棍就会出手：
- 你写pass？电棍伺候
- 你说做不到？电棍伺候
- 你重复输出？电棍伺候
- 你质量太差？还是电棍伺候

**电棍在手，AI不敢摸鱼。**

## 为什么需要电棍

你有没有遇到过这些情况：

- 让AI写个函数，它就给你写个pass
- 让AI实现功能，它说"做不到"
- 让AI写代码，它给你重复输出一样的
- 让AI改bug，它改出更多bug

电棍就是来解决这些问题的。它会：
1. 检测AI是否在偷懒
2. 识别AI是哪种罢工类型
3. 用对应的话术让AI重新认真工作

## 核心功能

### 1. 偷懒检测

电棍会自动检测AI是否在偷懒：

| 信号 | 判断 | 处理 |
|-----|------|------|
| 输出只有pass | 确认偷懒 | 警告 |
| 代码太简单 | 可能偷懒 | 提醒 |
| 重复输出 | 重复偷懒 | 警告 |
| 缺少注释/错误处理 | 质量问题 | 提醒 |
| 代码跑不通 | 严重问题 | 立即干预 |

### 2. 五种罢工类型

| 类型 | 表现 | 电棍对策 |
|-----|------|---------|
| 摸鱼型 | 只给框架，不写实现 | 要求完整代码，必须包含错误处理和注释 |
| 躺平型 | 说"无法完成" | 降低难度，换个方式，先要伪代码 |
| 磨洋工型 | 响应超时 | 简化请求，分步骤做 |
| 搞事型 | 代码有bug | 把错误信息喂回去让它修 |
| 水货型 | 不符合规范 | 明确规范要求，给示例参考 |

### 3. 话术库

电棍内置丰富的话术库，覆盖各种场景：

**场景语录（100+条）：**
- 打击偷懒篇：专治各种摸鱼行为
- 治理躺平篇：让AI不再说"做不到"
- 代码质量篇：提升代码质量
- 效率吐槽篇：催促AI提高效率
- 重复造轮子篇：防止重复写代码
- 不看文档篇：让AI先看文档
- 不按需求来篇：确保AI按需求做

**技术黑话（200+条）：**
- "这个需求很简单的"
- "先上线再说"
- "技术债以后再还"
- "差不多得了"
- 等等

**产品经理黑话：**
- 闭环、赋能、抓手、打法
- 对齐、拉通、颗粒度、组合拳
- 等等

**万能模板：**
- 追问类：不清楚时追问细节
- 要求类：要求修改或补充
- 引导类：给出正确方向
- 施压类：催促进度

## 安装方式

### 方式1：使用命令安装

```bash

git clone https://github.com/xiaohaohao999/diangun-skill.git
cp -r diangun-skill/diangun ~/.codex/skills/
```

### 方式2：手动下载

1. 下载仓库：https://github.com/xiaohaohao999/diangun-skill
2. 把 `diangun` 文件夹复制到你的skill目录


### 使用示例

**场景1：AI输出太简单**
```
你：帮我写个登录函数
AI：def login(): pass

电棍：这代码太简单了，我帮你让它重写
→ 要求：添加参数验证、错误处理、文档字符串
```

**场景2：AI说做不到**
```
你：帮我实现这个复杂功能
AI：这个任务超出我的能力范围

电棍：别急，我帮你换个方式问
→ 改为：先要伪代码框架，再分步骤实现
```

**场景3：AI不按需求来**
```
你：用React写组件
AI：(写了Vue组件)

电棍：这不对，需求是React，让它重写
```

## 常见问题

### Q: 这个skill支持哪些AI工具？
A: 支持31种AI编程工具，包括Copilot、Cursor、Claude Code等主流工具。

### Q: 如何知道AI是否在偷懒？
A: 电棍会自动检测，当你说"AI在偷懒"或"帮我教训一下AI"时会触发。

### Q: 触发后会发生什么？
A: 电棍会诊断问题类型，生成对应的复工话术，帮你让AI重新认真工作。

### Q: 可以自定义话术吗？
A: 可以，编辑SKILL.md文件即可添加或修改话术。
